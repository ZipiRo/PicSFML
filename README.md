# What is PicSFML?

PicSFML is a command-line toolkit designed to simplify the creation and management of SFML projects.

Creating a new SFML project usually requires configuring include directories, library directories, compiler options, copying DLLs, and creating a project structure before writing any code. PicSFML automates those tasks while keeping the project lightweight and easy to understand.

With PicSFML, creating and building a project can be as simple as:

```bash
picsfml create default MyProject
cd MyProject
picsfml build . release
```

A graphical interface is currently under development and will provide the same functionality without requiring the command line.

---

# Why PicSFML?

PicSFML aims to provide a better starting point for SFML development.

Instead of spending time configuring projects, copying libraries and creating the same folder structure for every new application, PicSFML lets you start writing code immediately.

It combines a project generator, build system and lightweight framework into a single tool while keeping projects simple and easy to customize.

# Features

- Create fully configured SFML projects.

- Configure projects entirely from the command line.

# Included Framework

Every project generated by PicSFML includes a lightweight framework that provides common functionality needed by most SFML applications.

The goal is to eliminate repetitive boilerplate code so you can focus on developing your application instead of setting up the project.

The framework currently includes:

## Application Class

The generated project includes an `Application` class that manages:

- Window creation
- Main game loop
- Event processing
- Update loop
- Rendering loop

The generated `main.cpp` already creates and runs this class, allowing you to begin development immediately.

## Resource Manager

The Resource Manager prevents resources from being loaded multiple times and provides centralized access to assets.

Currently supported resources include:

- Textures
- Fonts
- Shaders
- Sound Buffers

New resource types can easily be added.

## Input Manager

The Input Manager simplifies keyboard and mouse input.

Instead of manually processing every SFML event, helper functions are provided for common operations such as:

- Key pressed
- Key released
- Key held
- Mouse button states
- Mouse position

## Audio

The framework includes simple wrappers around SFML's audio module.

### Sound Player

Designed for short sound effects.

### Music Player

Designed for streamed background music.

Both classes simplify playback and resource management.

## Project Template

Every generated project already contains:

- Application class
- Build directories
- Resource directory
- Project configuration
- Application icon

allowing you to start writing your application immediately.

Resources directory is the default location for textures, fonts, audio, project assets.

# Requirements

PicSFML manages your projects, but **does not install external dependencies**.

Before using PicSFML you must install:

- A supported C++ compiler (currently MinGW GCC).
- SFML (currently versions 2.6.2 and 3.0.0).

After installing them, simply configure their locations once using the `defaults` command.

# First-Time Setup

Configure the compiler.

```bash
picsfml defaults compiler set "C:/mingw64/bin/g++.exe"
```

Configure the SFML installation.

```bash
picsfml defaults sfml set "C:/SFML-3.0.0"
```

Configure the SFML version.

```bash
picsfml defaults sfml_version set 3.0.0
```

Configure the default projects directory.

```bash
picsfml defaults projects_directory set "D:/Projects"
```

Once these values are configured, every new project created with

```bash
picsfml create default MyProject
```

will automatically use them.

---

# Selecting an SFML Version

PicSFML supports multiple versions of SFML.

Each project must specify which version it targets so PicSFML knows which framework layout, libraries and compiler options should be used.

Configure it globally:

```bash
picsfml defaults sfml_version set 3.0.0
```

or for an existing project:

```bash
picsfml config . sfml_version set 3.0.0
```

Currently supported versions:

- 2.6.2
- 3.0.0

If no SFML version is configured, PicSFML cannot determine which version of SFML should be used during the build.

# Creating Projects

Create a project at a specific location.

```bash
picsfml create "D:/Projects/MyGame" MyProejct
```

or create it inside your configured default projects directory.

```bash
picsfml create default MyProject
```

If your default projects directory is

```text
D:/Projects
```

then

```bash
picsfml create default MyProject
```

creates

```text
D:/Projects/
└── MyProject/
```

# Building Projects

Build the current project.

```bash
picsfml build .
```

Build a Release executable.

```bash
picsfml build . release
```

Build a Debug executable.

```bash
picsfml build . debug
```

---

# Project Configuration

Project settings are modified using the `config` command.

Show every configuration value.

```bash
picsfml config . list
```

Get the compiler path.

```bash
picsfml config . compiler get
```

Change the compiler.

```bash
picsfml config . compiler set "C:/mingw64/bin/g++.exe"
```

Restore the default compiler.

```bash
picsfml config . compiler default
```

Any property that supports the `default` command can be restored to the value configured through the `defaults` command.

---

# Global Defaults

The `defaults` command configures the values that every newly created project will inherit.

View every default value.

```bash
picsfml defaults list
```

Change the compiler.

```bash
picsfml defaults compiler set "C:/mingw64/bin/g++.exe"
```

Change the SFML installation.

```bash
picsfml defaults sfml set "C:/SFML-3.0.0"
```

Change the default SFML version.

```bash
picsfml defaults sfml_version set 3.0.0
```

Change the default projects directory.

```bash
picsfml defaults projects_directory set "D:/Projects"
```

Change the default main file name.

```bash
picsfml defaults main set "main.cpp"
```

Change the default output file name.

```bash
picsfml defaults output set "MPGrid"
```

Change the default build flags debug or release.

```bash
picsfml defaults flags debug set "-g"
```

---

# Default Values vs Project Values

The `defaults` command changes the global configuration.

The `config` command changes only the current project.

```text
                defaults
                    │
                    │
        ┌───────────┴───────────┐
        │                       │
 create default A         reate default B
        │                       │
     Project A              Project B
        │                       │
   config modifies        config modifies
    only Project A         only Project B
```

# Configurable Project Properties

The following properties can be modified inside a project.

| Property | Description |
|----------|-------------|
| `name` | Project name |
| `compiler` | Compiler executable |
| `sfml` | SFML installation path |
| `sfml_version` | Version of SFML used by the project |
| `app_version` | Application version |
| `output` | Output executable name |
| `main` | Main source file |
| `source` | Additional source files |
| `include` | Additional include files |
| `library` | Additional library files |
| `binary` | Additional binary/DLL files |
| `flags debug` | Debug compiler flags |
| `flags release` | Release compiler flags |
| `vscode` | Generate Visual Studio Code configuration |
| `audio` | Enable the SFML Audio module |
| `network` | Enable the SFML Network module |

> **Note**
>
> `picsfml_version` is maintained internally by PicSFML and should not be modified manually.

# The `.picsfml` File

Every project contains a `.picsfml` configuration file.

This file stores all project-specific settings used by PicSFML during project creation and building.

Although it can be edited manually, using the `config` command is recommended, as it validates values before saving them.

Example:

```text
name = MyProject

compiler = C:/mingw64/bin/g++.exe
sfml = C:/SFML-3.0.0
sfml_version = 3.0.0

output = MyProject
main = main.cpp

vscode = true
audio = false
network = false
```


---

# Project Icon

Every generated project contains an `icon.png` file.

To change the application's icon, simply replace this file with another PNG image while keeping the filename:

```text
icon.png
```

PicSFML automatically converts this image into the executable's icon during the build process.

No additional configuration is required.

# Example Workflow

Configure PicSFML.

```bash
picsfml defaults compiler set "C:/mingw64/bin/g++.exe"
picsfml defaults sfml set "C:/SFML-3.0.0"
picsfml defaults sfml_version set 3.0.0
picsfml defaults projects_directory set "D:/Projects"
```

Create a project.

```bash
picsfml create default MyProject
```

Open the project.

```bash
cd MyProject
```

Build it.

```bash
picsfml build . debug
```

If you later decide to use another compiler for only this project:

```bash
picsfml config . compiler set "D:/LLVM/bin/clang++.exe"
```

and later want to return to the default compiler:

```bash
picsfml config . compiler default
```
